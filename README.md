## 搬瓦工 KiwiVM 是什么？为什么不是 SolusVM？

**KiwiVM 是搬瓦工自主开发的 VPS 管理控制面板。**

大多数 VPS 商家用的是商业面板 SolusVM，功能标准化，但定制性差。搬瓦工从一开始就自己做了一套，叫 KiwiVM。功能上和 SolusVM 类似——重启、重装、改密码——但多了几个搬瓦工特有的东西，机房迁移是其中最受用的。

## OpenVZ vs KVM：一次该讲清楚的区别

很多老文章把这两个架构说得很复杂，其实用一句话概括就行：

**OpenVZ 是共享内核，KVM 是完全虚拟化。**

共享内核意味着什么？意味着你没有自己的独立内核，内核是和同一台宿主机上其他人共用的。这带来一个直接问题：你没办法自己换内核，也就没办法启用 BBR 加速（BBR 需要特定内核版本）。另外，OpenVZ 的系统选择也受限，最新版本的 Linux 系统在 OpenVZ 下可能跑不了。

KVM 就不一样了。每个 VPS 实例有独立内核，你可以随便改，BBR 一键开，内核随便换，装什么系统理论上都行（包括 Windows，前提是配置够）。稳定性上，OpenVZ 的一个风险是宿主机内核崩了，上面所有 VPS 实例一起挂；KVM 的隔离性更好，这种情况出现的概率小很多。

当年搬瓦工同时卖这两种，还是同价格。选 KVM 的理由碾压 OpenVZ，所以后来直接把 OpenVZ 清退掉了。现在的在售套餐，无论是入门的 KVM 套餐还是高端的香港 CN2 GIA，全是 KVM。

| 对比项 | OpenVZ（已下架） | KVM（当前在售） |
|--------|----------------|---------------|
| 内核 | 共享宿主机内核 | 独立内核 |
| BBR 加速 | 不支持 | 支持，可一键开启 |
| 系统选择 | 受限，新版 Linux 可能不兼容 | 全面支持，含最新 Debian/Ubuntu |
| 机房切换 | 不能切换到 CN2 优化机房 | 支持，含 CN2 GIA 等高速机房 |
| 稳定性 | 宿主机内核故障影响所有实例 | 隔离性更好 |
| 当前状态 | 2018 年底全面下架，不可续费 | 全部在售套餐均采用此架构 |

## KiwiVM 控制面板主要功能走一遍

买完搬瓦工，进 Client Area → My Services → 点 KiwiVM Control Panel，就进后台了。第一眼可能觉得界面有点简陋，但功能都在，用起来反而比那些大而全的面板省事。

### Main Controls（主控制页）

进去第一眼看到的就是这个。显示你的 VPS IP、SSH 端口、内存、硬盘、流量使用情况，以及开机/关机/重启按钮。

这里有个容易踩的坑：**SSH 默认端口不是 22**，是随机分配的，要在这个页面里查。很多新用户搞不清楚为什么 SSH 连不上，查了半天才发现端口搞错了。Root 密码也在这里生成。

### Install new OS（重装系统）

用的频率比较高。左边关机，然后进这个页面，选系统版本（CentOS、Debian、Ubuntu 都有），带 `_bbr` 后缀的是内置了 BBR 加速内核的版本。

说实话，现在新版 Debian 12 和 Ubuntu 22.04 都自带 BBR，不一定非要找带 bbr 后缀的，直接装新版发行版也行。

重装完成大概 1-2 分钟，页面会显示新的 root 密码和 SSH 端口，**同时也会发邮件通知你**。

### Migrate to another DC（迁移机房）

这是搬瓦工被用的最多的功能之一。

选目标机房，点 Confirm，然后 Start Migration，等通知邮件。迁移时间取决于你 VPS 里的文件量，轻则一分钟，重则几个小时。整个过程**不收费，次数没有限制**。

有几点要注意：

1. 切换机房会**换 IP**，如果有域名指向这个 IP，迁移完记得更新解析
2. **IP 已经被封的情况下无法迁移**，这是很多人以为可以靠换机房解封 IP 然后发现不行的原因
3. 部分机房切换过去后流量会变成原来的 1/3（标注了 0.33x 的机房）

### Snapshots（快照备份）

可以给当前 VPS 的完整状态拍一个快照。默认保存 30 天，设置成 Sticky 状态可以永久保留。

在重装系统或者迁移机房之前，养成先拍快照的习惯，万一出问题可以回滚。

### 其他实用功能

KiwiVM 面板还有几个不常用但关键时候救命的功能：

- **Root Shell**：在线 SSH，网页直接操作 VPS，本地没装 SSH 客户端的时候能用
- **Detailed Statistics**：查 CPU 使用率、带宽 I/O，诊断性能问题的时候看这里
- **API**：给有自动化需求的人用，可以通过 API 接口控制 VPS
- **Two-factor Authentication**：两步验证，在面板密码里设置，安全性要求高的可以开

## 常见问题

**Q：搬瓦工还有 OpenVZ 套餐卖吗？**

没有，2018 年底全面停售，老套餐也不让续费了。现在买到的全是 KVM 架构。

**Q：KiwiVM 面板怎么登录？**

进搬瓦工官网，点 Client Area 登录账号，然后去 Services → My Services，找到你的服务器，点 KiwiVM Control Panel 按钮就进去了。面板密码和购买账号密码是分开的，第一次进去需要单独设置或者用自动生成的密码。

**Q：搬瓦工 KiwiVM OpenVZ 方案的数据能不能迁移到 KVM？**

不能直接迁移，架构不同没办法直接升级。当时的做法是：申请退款（30 天内），然后重新购买 KVM 套餐。已经超期的 OpenVZ 套餐只能放弃，数据要自己手动迁移。

**Q：KiwiVM 切换机房要钱吗？**

不收费，次数也没限制。唯一要注意的是换机房会消耗流量（传输数据到新机房），以及 IP 会变。IP 被封的状态下无法操作切换机房。
